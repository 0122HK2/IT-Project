# **プロジェクト名称: シンプル掲示板アプリ（PHP版MVCモデル）**

## **1\. プロジェクト概要**

本プロジェクトは、PHPとMySQLを使用し、外部ライブラリ（Composer等）を一切使用せずにMVCアーキテクチャに基づいたシンプルな掲示板アプリケーションを構築する。

## **2\. 開発・実行環境**

* **OS**: Windows (XAMPP環境を想定)  
* **Webサーバー**: Apache  
* **言語**: PHP 8.x 以上  
* **データベース**: MySQL (MariaDB)  
* **フロントエンド**: HTML5, CSS3, JavaScript (Vanilla JS)  
* **文字コード**: UTF-8 (ファイルおよびデータベース)

## **3\. 機能要件**

### **3.1 投稿表示機能**

* データベースに保存されている投稿全件を取得し、一覧表示する。  
* 表示順序は投稿日時（created\_at）の降順（最新が一番上）とする。  
* 各投稿には「ニックネーム」「本文」「投稿日時」を表示する。

### **3.2 投稿保存機能**

* ユーザーが「ニックネーム」と「本文」を入力し、送信できる。  
* 投稿成功後、二重投稿防止のため「PRGパターン（Post-Redirect-Get）」を用いてトップページへリダイレクトする。  
* 投稿日時はサーバー側のシステム時間を自動取得する。

### **3.3 バリデーション機能**

* ニックネーム、本文ともに「未入力（空文字）」での投稿を許可しない。  
* サーバー側（PHP）でチェックを行い、不備がある場合はエラー処理を行う。

## **4\. 非機能要件・セキュリティ**

* **セキュリティ**:  
  * **XSS対策**: 表示時に htmlspecialchars を用い、すべての出力をエスケープする。  
  * **SQLインジェクション対策**: PDOのプリペアドステートメント（静的プレースホルダ）を必須とする。  
* **アーキテクチャ**:  
  * シンプルなMVCモデルを採用し、関心の分離（Model/View/Controller）を行う。  
  * 外部ライブラリは使用せず、PHP標準機能のみで実装する。

## **5\. データベース設計**

* **データベース名**: bbs\_db  
* **テーブル名**: posts

| カラム名 | 物理名 | データ型 | 制約 | 説明 |
| :---- | :---- | :---- | :---- | :---- |
| ID | id | INT | PRIMARY KEY, AUTO\_INCREMENT | 投稿ID |
| ニックネーム | nickname | VARCHAR(50) | NOT NULL | 投稿者の名前 |
| 本文 | message | TEXT | NOT NULL | 投稿内容 |
| 投稿日時 | created\_at | DATETIME | DEFAULT CURRENT\_TIMESTAMP | 投稿日時（自動生成） |

## **6\. ディレクトリ構成（推奨）**

simple_bbs/  
├── config/  
│   └── database.php       \# DB接続設定（PDO）  
├── controllers/  
│   └── PostController.php  \# 入力制御・モデルとビューの仲介  
├── models/  
│   └── Post.php           \# データ操作ロジック（SELECT/INSERT）  
├── views/  
│   └── index.php          \# 閲覧用UI（HTML/CSS/JS）  
└── index.php              \# エントリポイント（ルーターの役割）

## **7\. アプリケーション・フロー**

1. **ユーザーアクセス**: index.php へアクセス。  
2. **ルーティング**: PostController をインスタンス化。  
3. **データ取得**: PostController が Post モデルを呼び出し、全投稿を取得。  
4. **画面表示**: PostController が views/index.php を読み込み、取得データを渡す。  
5. **投稿実行**:  
   * ユーザーがフォーム送信（POST）。  
   * PostController がバリデーションを実行。  
   * 問題なければ Post モデルを介してDB保存。  
   * 保存完了後、index.php へリダイレクト。
